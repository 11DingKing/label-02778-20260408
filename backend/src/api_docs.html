🌐 人才网 API 接口文档

Rust Actix-web · /api · JWT Bearer · 点击接口展开详情

认证：请求头 Authorization: Bearer <token>，通过 POST /api/auth/login 获取。
响应格式：{"code":0,"message":"success","data":...}，错误时 code≠0。
分页：参数 page(默认1) size(默认10,最大50) keyword(搜索)；响应 {"list":[...],"total":N,"page":1,"size":10}
🔒 = 需登录 · 标注角色 = 需对应权限

🔑 Auth 认证 3
POST
/api/auth/register用户注册
▶
请求体 JSON
字段类型必填说明
usernamestring是2-50字符
passwordstring是6-50字符
roleint是0=求职者 1=企业
phonestring否手机号
请求示例
POST /api/auth/register
Content-Type: application/json

{"username":"张伟","password":"123456","role":0,"phone":"13800138000"}
响应示例
{"code":0,"message":"success","data":{"id":1,"username":"张伟","phone":"13800138000","avatar":"","role":0,"status":0,"created_at":"2026-01-01T00:00:00"}}

/api/auth/login用户登录，返回 JWT token
▶
请求体 JSON
字段类型必填说明
usernamestring是用户名
passwordstring是密码
响应示例
{"code":0,"message":"success","data":{"token":"eyJhbGci...","user":{"id":1,"username":"张伟","role":0,...}}}

/api/auth/me获取当前登录用户信息🔒 登录
▶
响应示例
{"code":0,"message":"success","data":{"id":1,"username":"张伟","phone":"138...","avatar":"","role":0,"status":0,"created_at":"..."}}

💬 Chat 聊天 6
POST
/api/chat/contact企业发起联系（每日限3人，可附招呼语）🔒 企业
▶
请求体 JSON
字段类型必填说明
seeker_user_idint是求职者用户ID
greetingstring否招呼语
请求示例
{"seeker_user_id":2,"greeting":"您好，我们对您的Java经验很感兴趣"}
业务规则

每个企业每天最多联系3名求职者（通过 SELECT FOR UPDATE 行锁保证并发安全）。超限返回 403。

PUT
/api/chat/contact/{id}/reply求职者回复（接受/拒绝）🔒 登录
▶
字段类型说明
acceptbooltrue=接受 false=拒绝
GET
/api/chat/contacts我的联系列表🔒 登录
▶
响应
{"code":0,"data":[{"id":1,"status":1,"other_username":"张伟","greeting":"您好...","unread_count":2,...}]}
POST
/api/chat/message发送消息（求职者回复自动接受联系）🔒 登录
▶
字段类型必填说明
contact_idint是联系ID
contentstring是消息内容
业务规则

求职者首次发送消息时，若联系状态为"待回复"，自动变为"已接受"。

GET
/api/chat/messages/{contact_id}获取聊天记录🔒 登录
▶
Query 参数
参数说明
page/size分页
PUT
/api/chat/messages/{contact_id}/read标记已读🔒 登录
▶

将指定联系的所有未读消息标记为已读。

🌍 Jianghu 江湖说 6
POST
/api/jianghu/post发布动态（所有人可发布，含匿名）
▶
认证

可选。登录用户关联身份，未登录以匿名身份发布。

请求体 JSON
字段类型必填说明
contentstring是动态内容，不能为空
imagesstring[]否图片URL数组（需先调用上传接口）
请求示例
POST /api/jianghu/post
Content-Type: application/json

{"content":"今天面试了一家互联网公司，感觉不错！","images":["/uploads/20260226_abc.png"]}
响应示例
{"code":0,"data":{"id":7,"user_id":null,"content":"今天面试了...","images":"","like_count":0,"comment_count":0,"created_at":"..."}}
GET
/api/jianghu/posts动态列表（无需登录，登录后显示点赞状态）
▶
Query 参数
参数说明
page/size分页
响应字段
字段说明
username发布者用户名，匿名发布显示"匿名用户"
liked当前用户是否已点赞（未登录始终false）
POST
/api/jianghu/post/{id}/like点赞/取消点赞🔒 登录
▶
响应
{"code":0,"data":true}  // true=已点赞, false=已取消
POST
/api/jianghu/post/{id}/comment发表评论🔒 登录
▶
字段类型必填说明
contentstring是评论内容
GET
/api/jianghu/post/{id}/comments评论列表
▶
响应
{"code":0,"data":[{"id":1,"user_id":2,"content":"说得好！","username":"张伟","avatar":"","created_at":"..."}]}
DELETE
/api/jianghu/post/{id}删除动态（本人或管理员）🔒 登录
▶

仅动态发布者本人或管理员可删除。匿名动态仅管理员可删除。
